TDD Rule을 작성해줘(프로젝트 스코프로 생성해)

## 반드시 포함할 내용

1. **적용 기준 구분**
   - TDD를 반드시 적용해야 하는 대상 목록
   - TDD가 불필요한 대상 목록
   - (비즈니스 로직, API, 유틸, 버그수정 → 적용 / 타입정의, 설정파일, 순수 UI, SQL → 불필요)

2. **RED-GREEN-REFACTOR 사이클** 각 단계별 세부 지침
   - RED: 하나의 동작 = 하나의 테스트, 반드시 실행해서 실패 확인, 실패 이유가 "기능 미구현"이어야 함
   - GREEN: 최소한의 코드만 작성, YAGNI 원칙, 신규+기존 테스트 모두 통과 확인
   - REFACTOR: 중복 제거/이름 개선/헬퍼 추출, green 유지, 새 동작 추가 금지
   - 반복: 다음 동작에 대한 RED로 돌아가기

3. **삭제 강제 규칙**
   - 테스트 전에 프로덕션 코드를 먼저 작성했다면 삭제하고 RED부터 재시작
   - "참고용"으로 남기는 것도 금지

4. **변명 차단표** (마크다운 테이블)
   - "너무 단순해서 테스트 불필요", "나중에 추가하겠다", "시간이 없다",
     "삭제하면 낭비", "프로토타입이다" 각각에 대한 반론

5. **프로젝트별 규칙 우선 조항**
   - 프로젝트의 CLAUDE.md/AGENTS.md에 TDD 규칙이 있으면 그걸 우선
   - 이 파일은 기본값(fallback)임을 명시

## 톤 & 스타일

- 파일 상단에 "이 규칙은 Rigid — 상황에 맞게 변형하지 마라" 경고 명시
- 강조할 항목은 **볼드** 처리
